# init.sh — Two-Tier Verification

The `init.sh` generated by `loom init` is a two-tier script: quick mode for fast dev feedback (<10s), full mode for the closeout gate before marking a feature done. This doc covers both tiers, customization, and the companion `scripts/verify-quick.sh`.

## 1. Two-Tier Mode

```bash
./init.sh              # full verification (closeout)
./init.sh quick        # dev cycle (lint + smart test subset)
MODE=quick ./init.sh   # env var alternative
```

Full mode runs the complete pipeline: lint, type check, all tests, build. Pass this before marking a feature `done`. Quick mode runs static checks and a fast test subset (no snapshot/slow tests). Use quick after every change during development.

## 2. What Quick Mode Does

```
=== Harness Initialization (quick) ===
=== ruff check . ===
=== mypy . ===
=== python -m pytest -x -q -m 'not slow and not snapshot' --tb=short ===
```

**Static checks are conditional.** `loom init` inspects `pyproject.toml` for `[tool.ruff]` / `[tool.mypy]` sections. Only detected tools appear in both tiers. If neither section exists, only the pytest command runs.

**Pytest smart subset:**

- `-x` stops at the first failure. No wasted cycles.
- `-m 'not slow and not snapshot'` skips long-running and visual-snapshot tests.
- `--tb=short` gives a 3-line traceback, enough to see the assertion.

## 3. Custom Commands

Override at scaffold time:

```bash
loom init --commands "ruff check ., mypy ., python -m pytest"
```

This sets both tiers to the same command list (comma-separated). To customize after scaffold, edit the `case "$MODE" in` block in `init.sh`:

```bash
case "$MODE" in
  quick)
echo "=== ruff check . ==="
ruff check .
echo "=== python -m pytest -x -q -m 'not slow and not snapshot' --tb=short ==="
python -m pytest -x -q -m 'not slow and not snapshot' --tb=short
    ;;
  full)
...same commands, no -x / -m flags...
    ;;
esac
```

Changes apply to one or both tiers. Full omits the test exclusions so every test runs.

## 4. scripts/verify-quick.sh

Standalone dev-cycle file, same purpose as `./init.sh quick` without the extra argument:

```bash
scripts/verify-quick.sh                      # auto-infer scope from git diff
scripts/verify-quick.sh tests/test_tools.py  # run a specific file
scripts/verify-quick.sh --no-skip-snapshot   # include snapshot tests
EVAL_FILTER=read-file scripts/verify-quick.sh
```

**No arguments** auto-infers scope via `git diff --name-only HEAD`. Change `loom/agent/context.py` and only `tests/test_context.py` runs. **`test_file.py`** runs that file only. **`--no-skip-snapshot`** includes `snapshot`-marked tests. **`EVAL_FILTER=`** also runs `loom eval --filter <name>`.

Without a git repo, or when git diff is empty, the script runs a smoke set instead of narrowing. It never errors.

Generated version in user projects is simpler than loom's own (no eval-case extraction or untracked-file logic).

## 5. Pytest Marker Configuration

`loom init` injects these markers into `pyproject.toml` under `[tool.pytest.ini_options]` if missing:

```toml
[tool.pytest.ini_options]
markers = [
    "slow: tests that take >5s; skipped via -m 'not slow'",
    "snapshot: visual/terminal-snapshot via snap_compare",
    "integration: tests needing external services or real LLM calls",
]
```

Add markers to your tests:

```python
@pytest.mark.slow
def test_heavy_computation():
    ...
```

Run with exclusion:

```bash
pytest -m 'not slow and not snapshot'    # dev cycle
pytest -m 'not snapshot'                  # all but visual
pytest -m 'slow'                          # only slow ones
```

The `snapshot` marker is auto-applied by `conftest.py` to `snap_compare` tests. No decorator needed.

## 6. Troubleshooting

**Non-git project falls back to smoke tests.** `scripts/verify-quick.sh` uses `git diff` for auto-scope. In a non-git directory, `git diff` fails silently and the script runs a fixed smoke set instead. Harmless, just not narrowed.

**"WARNING: unknown marker 'snapshot'" from pytest.** The marker is missing from `[tool.pytest.ini_options] markers`. Tests still run. Fix by adding the marker or re-running `loom init --force`.

**"command not found: ruff" / "command not found: mypy".** These tools are in the script because pyproject.toml has `[tool.ruff]` / `[tool.mypy]`, but the binary is not installed. Install with `pip install ruff mypy`, or remove the lines from `init.sh`.

**"syntax error: unexpected token" on bash 3.x.** Generated scripts use `[[ ]]`, `case`, and arrays that need bash 4+. macOS ships bash 3.2. Install a newer one: `brew install bash`, then run with `/opt/homebrew/bin/bash init.sh`.

**Quick mode runs full tests.** The `-m 'not slow and not snapshot'` flag is missing from the `quick` case in `init.sh`. An edit may have removed it; add it back.

## 7. Unix-Only Notes

- Windows is **not supported natively**. Scripts use bash features (`[[ ]]`, `case`, arrays) that do not work under `cmd.exe` or PowerShell.
- Use **WSL** or **Git Bash** on Windows.
- Scripts assume POSIX paths and Unix line endings. CRLF endings cause `"\r: command not found"` errors.
- `init.sh` and `scripts/verify-quick.sh` are created executable by `loom init`. Copying them manually requires `chmod +x`.
